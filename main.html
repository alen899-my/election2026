Kerala 2026 — Legislative Assembly Election Platform

A comprehensive, open-source web platform dedicated to the 2026 Kerala Legislative Assembly elections. Built to provide voters, journalists, researchers, and political enthusiasts with beautiful, data-rich profiles of all 140 constituencies, their candidates, historical voting patterns, and real-time election results.

Mission & Vision

Kerala 2026 is built on the belief that transparent, accessible election information strengthens democracy. Every voter in Kerala deserves to know exactly who is standing for election in their constituency, what their track record looks like, and how they have voted or acted in the past. This platform makes that information beautiful, searchable, and shareable.

As an open-source project, Kerala 2026 invites developers, data journalists, designers, and civic technologists to contribute. The code is publicly available, the data is openly licensed where possible, and the platform is free to use for anyone.

Why this matters: The 2021 Kerala elections saw a voter turnout of 74.06%. With better access to candidate information, we aim to help voters make more informed choices in 2026 — and to hold elected representatives accountable with publicly accessible historical data.

Kerala Election Context

The Kerala Legislative Assembly (Niyamasabha) consists of 140 elected members from 140 single-member constituencies. Elections are held every 5 years. The 2021 election saw LDF (Left Democratic Front) win 99 seats, making history by being the first ruling coalition to win back-to-back in Kerala in 40 years.

Project Roadmap

Project Phases

Kerala 2026 is built in two distinct phases, each with clear deliverables and timelines. Phase 1 launches before the election with comprehensive candidate information. Phase 2 activates on result day with live data.

1
Phase 1 — Pre-Election Platform (Current)
Build the complete candidate and constituency information platform. This phase focuses on creating beautiful, data-rich profiles for all candidates, comprehensive constituency pages, historical election data visualization, and party analysis dashboards. The platform should be live at least 60 days before election day to give voters maximum time to research candidates.
Candidate Profiles Constituency Pages Historical Data Party Dashboards Search & Filter Static Site
2
Phase 2 — Live Election Results (Result Day)
On election result day, activate a live data pipeline that scrapes official results from the Kerala CEO website (ceo.kerala.gov.in) and Election Commission of India portal. Results update in real-time, showing leads, vote counts, round-by-round updates, and trend analysis. The scraper runs every 60 seconds during counting. A WebSocket layer pushes updates to all connected browsers without page refresh.
Live Scraping WebSocket Updates Real-time Results Lead Tracker Auto-refresh
3
Phase 3 — Post-Election Archive (Future)
After results are declared, the platform transitions to a permanent archive — the definitive source for 2026 Kerala election data. Includes winner profiles, margin analysis, swing analysis, constituency-level breakdowns, and integration of 2026 data into the historical trend visualizations. This phase sets the foundation for future election cycles.
Archive Mode Winner Profiles Swing Analysis Future Planning

Technical Decisions

Tech Stack

Kerala 2026 uses a carefully chosen set of modern, open-source technologies optimized for performance, developer experience, and long-term maintainability. Every choice is justified by the project's specific needs.

Full Stack Overview

Frontend Framework
Next.js 14
App Router with SSG for constituency pages. ISR for data updates. SEO-optimized static generation for all 140+ pages.
Styling
Tailwind CSS
Custom design system with Kerala-inspired color palette. Fully responsive, mobile-first approach.
Language
TypeScript
Full type safety across frontend and backend. Critical for data integrity with complex election datasets.
Backend
Node.js + Express
REST API for candidate/constituency data. WebSocket server for Phase 2 live results streaming.
Database
PostgreSQL (Neon)
Serverless PostgreSQL for all structured data. Drizzle ORM for type-safe queries. Full-text search with pg_trgm.
ORM
Drizzle ORM
Lightweight, type-safe SQL. Schema migrations. Perfect for complex relational election data.
Image Storage
Cloudflare R2
Candidate photos stored in R2. CDN delivery globally. Zero egress fees for open-source scale.
Search
Meilisearch
Fast full-text search across 1000+ candidates, constituencies, and parties. Malayalam + English support.
Charts
Recharts + D3.js
Interactive vote share charts, historical trends, constituency maps. SVG-based, fully accessible.
Maps
Mapbox GL JS
Kerala constituency map with clickable regions. Color-coded by party, turnout, and winning margin.
Scraping (Phase 2)
Puppeteer + Cheerio
Puppeteer for JavaScript-rendered pages. Cheerio for static HTML parsing. Handles Kerala CEO website structure.
Real-time (Phase 2)
Socket.io
WebSocket connections for live result updates. Broadcasts new data to all connected clients every 60 seconds.
Deployment
Vercel + Railway
Frontend on Vercel Edge Network. Backend on Railway with auto-scaling. Global CDN for static assets.
Analytics
Plausible
Privacy-friendly analytics. No cookies. Open source. Understand visitor patterns without compromising user privacy.
CMS / Admin
Custom Admin UI
Internal data entry tool for constituency coordinators to add and verify candidate data before publication.
Email
Resend
Contributor notifications, data correction alerts, and election day result digest emails.

Why These Choices?

TechnologyAlternative ConsideredWhy We Chose This
Next.js 14Remix, AstroSSG + ISR combo is perfect for mostly-static constituency pages that update occasionally. SEO is critical for election search traffic.
PostgreSQL (Neon)Supabase, MongoDBRelational data is perfect for election datasets. Complex joins needed (candidate → constituency → party → results). Neon serverless scales to zero.
Drizzle ORMPrismaLighter bundle, better TypeScript inference, runs in edge environments. Important for Next.js API routes.
MeilisearchAlgolia, ElasticsearchOpen source, self-hostable, excellent multilingual support. Malayalam search is critical. No per-search pricing.
Mapbox GL JSGoogle Maps, LeafletExcellent Kerala constituency GeoJSON support. Custom styling to match design. Reasonable free tier for open-source.
Cloudflare R2AWS S3, Supabase StorageZero egress fees is critical for open-source scale. 140+ constituency images, 1000+ candidate photos adds up.

Database Schema

The database is designed around three core entities: constituencies, candidates, and election results. All tables use UUID primary keys. The schema supports both Phase 1 (static data) and Phase 2 (live results) requirements.

Schema Note: All tables include created_at and updated_at timestamps. Soft deletes via deleted_at. The schema is versioned and managed via Drizzle Kit migrations.

Drizzle Schema Example

TypeScript — packages/db/schema/candidates.ts
copy
import { pgTable, uuid, varchar, text, date, boolean, integer, bigint,
         timestamp, pgEnum } from 'drizzle-orm/pg-core';
import { constituencies } from './constituencies';
import { parties } from './parties';

export const genderEnum = pgEnum('gender', ['male', 'female', 'other']);
export const nominationEnum = pgEnum('nomination_status', ['filed', 'accepted', 'withdrawn', 'rejected']);

export const candidates = pgTable('candidates', {
  id:                   uuid('id').defaultRandom().primaryKey(),
  constituencyId:       uuid('constituency_id').notNull().references(() => constituencies.id),
  partyId:              uuid('party_id').references(() => parties.id),
  nameEn:               varchar('name_en', { length: 200 }).notNull(),
  nameMl:               varchar('name_ml', { length: 200 }),
  slug:                 varchar('slug', { length: 100 }).notNull().unique(),
  photoUrl:             text('photo_url'),
  dateOfBirth:          date('date_of_birth'),
  gender:               genderEnum('gender'),
  religion:             varchar('religion', { length: 50 }),
  education:            varchar('education', { length: 200 }),
  profession:           varchar('profession', { length: 200 }),
  isIncumbent:          boolean('is_incumbent').default(false),
  termsServed:          integer('terms_served').default(0),
  criminalCases:        integer('criminal_cases').default(0),
  totalAssetsInr:       bigint('total_assets_inr', { mode: 'number' }),
  totalLiabilitiesInr:  bigint('total_liabilities_inr', { mode: 'number' }),
  affidavitUrl:         text('affidavit_url'),
  bio:                  text('bio'),
  electionYear:         integer('election_year').default(2026),
  nominationStatus:     nominationEnum('nomination_status').default('filed'),
  createdAt:            timestamp('created_at', { withTimezone: true }).defaultNow(),
  updatedAt:            timestamp('updated_at', { withTimezone: true }).defaultNow(),
});

export type Candidate = typeof candidates.$inferSelect;
export type NewCandidate = typeof candidates.$inferInsert;

Developer Setup

Environment Setup

Complete guide to setting up Kerala 2026 for local development. Takes approximately 20 minutes for a fresh setup.

Prerequisites

Required: Node.js 20 LTS, pnpm 9+, Git, a Neon account (free), Cloudflare account (free tier). Optional: Docker for local PostgreSQL instead of Neon.

Quick Start

Shell — initial setup
copy
# Clone the repository
git clone https://github.com/kerala-2026/kerala-2026.git
cd kerala-2026

# Install pnpm if not already installed
npm install -g pnpm@9

# Install all dependencies
pnpm install

# Set up environment files
cp apps/web/.env.example apps/web/.env.local
cp apps/api/.env.example apps/api/.env

# Fill in your values in the .env files (see below)

# Run database migrations
pnpm --filter @kerala2026/db db:generate
pnpm --filter @kerala2026/db db:migrate

# Seed with sample data (14 districts, 140 constituencies)
pnpm --filter @kerala2026/api db:seed

# Start all apps in development mode
pnpm dev

# Apps will be available at:
# Web:   http://localhost:3000
# API:   http://localhost:3001
# Admin: http://localhost:3002
