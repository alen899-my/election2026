Layer 1 — Shared Package

DB Schema Module

packages/db — The foundation of the entire project. Defines all Drizzle ORM table schemas, enums, relations, and exports TypeScript types. Must be built and seeded before any other module can function.

Files: packages/db/schema/ · packages/db/migrations/ · packages/db/seed/ · drizzle.config.ts
packages/db
Database Schema & Migrations
Drizzle ORM table definitions, enums, seed data, migration files
P0 — Build First
Schema Files — Create Each Table
Create schema/districts.ts — districts table
Fields: id (uuid PK), name_en, name_ml, slug (unique), headquarters, constituency_count, geojson_url, created_at, updated_at
P0
Create schema/constituencies.ts — constituencies table with categoryEnum
Fields: id, district_id (FK → districts), number (unique 1-140), name_en, name_ml, slug (unique), category (general/sc/st), total_voters_2021, total_voters_2026, male_voters, female_voters, geojson (jsonb), description, created_at, updated_at, deleted_at. Export Constituency + NewConstituency types.
P0
Create schema/parties.ts — parties table with coalitionEnum
Fields: id, name_en, name_ml, abbreviation, slug (unique), coalition (LDF/UDF/NDA/IND enum), color_hex, logo_url, founded_year, ideology, website_url, created_at, updated_at
P0
Create schema/candidates.ts — candidates table with genderEnum + nominationEnum
Fields: id, constituency_id (FK), party_id (FK nullable), name_en, name_ml, slug (unique), photo_url, date_of_birth, gender, religion, caste, education, profession, is_incumbent (bool), terms_served (int), criminal_cases (int), total_assets_inr (bigint), total_liabilities_inr (bigint), affidavit_url, bio, social_facebook, social_twitter, social_instagram, election_year (default 2026), nomination_status, created_at, updated_at, deleted_at
P0
Create schema/election_results.ts — historical + 2026 results
Fields: id, constituency_id (FK), candidate_id (FK nullable — for historical data candidate may not be in DB), party_id (FK nullable), election_year, candidate_name (varchar — for historical records), votes_received, vote_percentage (decimal 5,2), position, is_winner (bool), winning_margin, total_votes_polled, voter_turnout_pct (decimal 5,2), nota_votes, created_at
P0
Create schema/candidate_assets.ts — detailed asset breakdown
Fields: id, candidate_id (FK), asset_type (movable/immovable enum), description, value_inr (bigint), source (varchar). Separate table to allow listing multiple assets per candidate.
P1
Create schema/criminal_cases.ts — per-case criminal records
Fields: id, candidate_id (FK), case_number, section (text — IPC sections), court, status (varchar), is_serious (bool), year. candidates.criminal_cases is a denormalized count for quick filtering.
P1
Create schema/live_results.ts — Phase 2 counting data
Fields: id, constituency_id (FK), candidate_id (FK), votes_so_far, rounds_counted, total_rounds, is_leading (bool), lead_margin, status (counting/declared/leading enum), last_updated (timestamptz). Upserted every 60s during counting.
Phase 2
Create schema/index.ts — barrel export of all tables
Export * from all schema files. Used by both the API and the seed script via @kerala2026/db workspace import.
P0
Migration & Config
Create drizzle.config.ts — Drizzle Kit configuration
Set schema path to ./schema/index.ts, output to ./migrations, dialect to postgresql, read DATABASE_URL from env. Enable verbose + strict mode.
P0
Generate initial migration with pnpm db:generate
Run after all schema files are created. Commit the generated SQL migration files to git. Every future schema change must go through db:generate → db:migrate.
P0
Create indexes.sql — performance indexes for production
GIN trigram indexes on candidates.name_en and name_ml for ILIKE search. Composite index on (constituency_id, election_year) for fast per-constituency candidate queries. Index on election_results(constituency_id, election_year). Index on live_results(constituency_id, last_updated DESC).
P1
Seed Data
Create seed/data/districts.ts — all 14 Kerala districts
All 14 districts with nameEn, nameMl (Malayalam script), slug, headquarters, and exact constituency count: Thiruvananthapuram (11), Kollam (10), Pathanamthitta (5), Alappuzha (9), Kottayam (9), Idukki (5), Ernakulam (14), Thrissur (13), Palakkad (12), Malappuram (13), Kozhikode (12), Wayanad (3), Kannur (11), Kasaragod (5). Total = 140.
P0
Create seed/data/constituencies.ts — all 140 constituencies
All 140 constituencies with official number (1-140), nameEn, nameMl (verified Malayalam), slug, districtSlug (for FK lookup), category (SC constituencies include Nedumangad, Aroor, Devikulam etc.). Source: Kerala Assembly official list from assembly.kerala.gov.in.
P0
Create seed/data/parties.ts — major parties of Kerala
At minimum: CPM (LDF), CPI (LDF), INC (UDF), IUML (UDF), KC(M) (UDF), BJP (NDA), BDJS (NDA), NCP (varies), RSP (LDF), JD(S). Include name_en, name_ml, abbreviation, coalition, color_hex, founded_year.
P0
Create seed/seed.ts — main seed runner
Insert districts first, build slug→id map, insert constituencies with correct district_id FK, insert parties. Use .onConflictDoNothing() so seed is idempotent (safe to run multiple times). Log counts after each step.
P0
Create seed/data/historical_results.ts — election results 1957–2021
This is significant data entry work. Source from ECI historical data. At minimum: 2021 and 2016 results for all 140 constituencies (winner name, party, votes, margin, turnout). Full 1957-2021 dataset is P1 — needed for trend charts.
P1

P0
Type Definition Files
Define base entity types from Drizzle schema (Constituency, Candidate, Party, ElectionResult)

Define relational types: CandidateWithParty, CandidateWithConstituency, CandidateFull

Define ConstituencyFull type — constituency with candidates, historical results, district
Used by the constituency detail page. Includes candidates: CandidateWithParty[], historicalResults: ElectionResult[], district: District.
P0

P0
Define search result types: SearchResults, CandidateSearchHit, ConstituencySearchHit
Meilisearch results have _formatted (highlighted) fields. Types must include optional _formatted nested object for highlighting support in SearchBar UI.

Converts "Pinarayi Vijayan" + "dharmadom" → "pinarayi-vijayan-dharmadom". Strips special chars, lowercases, replaces spaces with hyphens. Must handle names with punctuation (K. Muraleedharan → k-muraleedharan).

Create formatVoteCount(n: number) — Indian number formatting
Uses Indian numbering system: 1,23,456 (not 123,456). Node: use toLocaleString('en-IN').

Create getPartyColor(coalition: string) and getCoalitionLabel(coalition: string)
Maps coalition codes to hex colors (LDF→#E31E24, UDF→#00BFFF, NDA→#FF6B35, IND→#888888) and display labels. Used in charts, map fills, and party badges.
P0
Create calculateElectionCountdown(electionDate: Date)
Returns { days, hours, minutes, seconds, isOver: bool }. Used in the home page countdown timer component. Update the election date constant once announced.
P1
Create cn(...classes) — Tailwind class merging helper
Wrapper around clsx + tailwind-merge. Prevents conflicting Tailwind classes when conditionally applying styles. Used extensively in every component.
P0
Create truncate(str: string, max: number) — text truncation with ellipsis
Truncates at word boundary, not mid-word. Used for candidate bios in card views.

modules/auth
Authentication Module
JWT admin login, requireAdmin middleware, token generation
P0
Create auth.service.ts — adminLogin function
bcrypt.compare password against ADMIN_PASSWORD_HASH from env. On success: jwt.sign({ role: 'admin' }, ADMIN_JWT_SECRET, expiresIn: '12h'). On failure: throw AppError(401). Include utility comment for generating ADMIN_PASSWORD_HASH: node -e "require('bcryptjs').hash('pass', 12).then(console.log)"
P0
Create middleware/auth.ts — requireAdmin Express middleware
Read Authorization: Bearer header. Verify JWT with ADMIN_JWT_SECRET. Check payload.role === 'admin'. Handle TokenExpiredError → 401 TOKEN_EXPIRED. Handle invalid → 401. Export requireAdmin function.
P0
Create auth routes: POST /api/v1/auth/login and GET /api/v1/auth/verify
POST /login: accepts { password }, returns { token, expiresIn: '12h' }. GET /verify: requires Bearer JWT, returns { valid: true, role }. Both return JSON error on failure.
